How to Create a Blog Using Python, GitHub, Pelican, and Jupyter Notebook

Date Sun 10 December 2017 Tags Python / GitHub / Pelican / Jupyter Notebook

Congratulations that you finally decided to take the first step of creating a blog!

In this guide, I will walk you through step-by-step how to create a blog on GitHub.io. The good news is that all the tools used in this process are intuitive and free. At the end of the guide, you should be able to:

  • Create a blog on GitHub.io
  • Set up the initial look and feel for the blog using Pelican Bootstrap3 theme
  • Write your first post using Jupyter Notebook and Markdown

I'm using Windows to demonstrate the steps. There could be some differences if you use a different OS but the general steps should be quite similar.

Prerequisites¶

Before we get into the business of creating a blog, let's make sure you have a couple of things ready:

  • Python 3 or Python 2.7 set up on your computer (I have Python 3 on my computer but it's fine if you prefer Python 2.7)
  • A GitHub account

If you have both, feel free to skip this section and move to Virtual Environment Setup.

If you don't have Python 3 or Python 2.7 on your computer, I highly recommend you to use Anaconda Python installation, especially if you work or are interested in working in the field of data science. Anaconda Python installation covers hundreds of packages that you'll find useful when doing analytics. It also includes Jupyter Notebook installatioin which we'll use to build your first blog post.

Anaconda Python installation is pretty painless, you need to:

  • Download the installer from this website
  • Run the .exe downloaded and follow instructions on the screen

I installed Anaconda Python distribution to the default path C:\Users\Dan\Anaconda3. The version of installer you should download depends on:

  • Whether your computer is 64-Bit or 32-Bit (mine is 64-Bit)
  • Whether you prefer to install Python 3 or Python 2.7 (I installed Python 3)

Once Python installation is completed, double check whether the installation is successful by executing conda info --all in the Windows command shell.

If you're not familiar windows command shell, you basically need to:

  1. Click on the Windows start button
  2. Enter CMD in the search box and hit enter
  3. Click on Command Prompt in the search results and a black window would pop up
  4. Enter conda info --all into the black window and hit enter

Regarding the GitHub account, if you don't have one, just go to GitHub and sign it up using your email address. It's quite straightforward.

Set Up a Virtual Environment Setup¶

Virtual environment is a tool to create isolated environment independent of the main Python distribution on your computer. Each time when you start a Python project, it's recommended to create an isolated environment for the project, so that you can avoid problem of one project requring version 123 of a Python library but another project requiring version 456 instead. Setting up your blog is a new project and you should set up a virtual environment just for it.

  • With Anaconda Python installation completed, you can now use conda to set up a virtual environment by executing conda create -n yourenvname python=x.x anaconda in the Windows command shell. yourenvname can be replaced with anything that you'd like to name your virtual environment. In my case, I called my virtual environment pelicantest.

  • Upon executing conda create -n yourenvname python=x.x anaconda, you'll be prompted to confirm whether you'd like to proceed. Press y to proceed.

  • To activate your virtual environment, execute activate yourenvname in the Windows command shell.

C:\Users\Dan>activate pelicantest

And you should see (yourenvname) appears as the leading string in the next line in the Windows command shell.

(pelicantest) C:\Users\Dan>

Upon successfully set up the virtual environment, a new folder pelicantest appears under C:\Users\Dan\Anaconda3\envs\.

Some of you may have a different Python distribution other than Anaconda. In that case, you can set up virtualenv and activate it by following instructions here.

Install Pelican and Python Packages¶

Pelican is one of the most popular static site generators. Static site generators free you from manually editing HTML files. With their help, you can focus on:

  • Create and edit content of your website in extremely simple format, e.g. in Markdown or other markup languages
  • Manage and update the theme of your website by coding the template files

Static site generators can automatically convert your Markdown files and template files into HTML. Isn't that great?!

There are a few popular static site generators:

  • Jekyll written in Ruby
  • Pelican written in Python
  • Metalsmith written in JavaScript

Since this guide is about using Python to create the website, we pick Pelican as our static site generator. With Pelican, we can also enable Jupyter Notebook integration by adding it as a plug-in which we'll discuss it later.

If you're interested in comparing pros and cons of these three static site genertors, this link provides more details.

Now follow the steps below to install Pelican and other Python packages to your Python virtual environment.

  • Create a folder on your computer to store content and templates of your website. Name it anything you'd like. In my case, I created a folder TDSblog on D:\ drive.

  • Enter your newly created folder using following command in the Windows command shell. Note that my virtual environment is active since its names leads the command line.

(pelicantest) C:\Users\Dan>pushd D:\TDSblog
  • Upon entering your new folder, you should see following in your Windows command shell
(pelicantest) D:\TDSblog>

  • Create a file .gitignore with content on this page and save .gitignore in your new folder.

    It tells Git to ignore files of certain types when pushing content from local computer to remote repository later. Files that we want Git to ignore are usually those automated generated log files and build files etc. This link provides more background about .gitignore if you're interested.

  • Create a file requirements.txt and save it to your folder C:\Users\Dan\Anaconda3\envs\pelicantest.

    Anaconda Python installation includes over 100 pre-built packages. So if you installed Anaconda Python distribution, copy and paste the following content into requirements.txt. They are Python packages that you'll find useful when buidling websites but may be not included in Anaconda Python installation.

beautifulsoup4~=4.6.0

ghp-import~=0.5.5

gitpython~=2.1.5

html5lib~=0.999999999

Markdown~=2.6.8

Pygments~=2.2.0

pyreadline~=2.1

smartypants~=2.0.0

typogrify~=2.0.7

If you a different Python distribution other than Anaconda, copy and paste the following contents to requirements.txt instead, in case your installation doesn't cover these useful pacakges for website building.

beautifulsoup4~=4.6.0

Fabric3~=1.13.1.post1

feedgenerator~=1.9

ghp-import~=0.5.5

gitpython~2.1.5

html5lib~=0.999999999

jupyter~=1.0.0

ipython~=6.1.0

Markdown~=2.6.8

matplotlib~=2.0.2

nbconvert~=5.2.1

pelican~=3.7.1

Pygments~=2.2.0

pyreadline~=2.1

smartypants~=2.0.0

typogrify~=2.0.7

If you're interested in the latest version of these Python packages, go to this site and search for the package you'd like to update. It also explains what a package is for.

  • Execute the following command in Windows command shell to install Python packages listed in requirements.txt
(pelicantest) D:\TDSblog>pip install -r C:\Users\Dan\Anaconda3\envs\pelicantest\requirements.txt

To double check whether the installation is successful, execute the following commands. It'll show all the Python packages installed so far and you should be able to find those listed in requirements.txt.

(pelicantest) D:\TDSblog>pip list

Create a Default Website¶

With the virtual environment set up and Python packages installed, we can now start setting up the website in the default style.

  • Execute the the following command to start the initial setup of your website
(pelicantest) D:\TDSblog>pelican-quickstart
  • The following questions will pop up in the Windows command shell. For the initial setup, you can press enter for majority of questions, except the ones asking you about the website title, author name, default language, whether to specify a URL prefix, and time zone.
Where do you want to create your new web site? [.]

What will be the title of this web site? Superduper Blog

Who will be the author of this web site? Superduper Writer

What will be the default language of this web site? [en]

Do you want to specify a URL prefix? e.g., http://example.com   (Y/n) n

Do you want to enable article pagination? (Y/n)

How many articles per page do you want? [10]

What is your time zone? [Europe/Paris] America/New York

Do you want to generate a Fabfile/Makefile to automate generation and publishing? (Y/n)

Do you want an auto-reload & simpleHTTP script to assist with theme and site development? (Y/n)

Do you want to upload your website using FTP? (y/N)

Do you want to upload your website using SSH? (y/N)

Do you want to upload your website using Dropbox? (y/N)

Do you want to upload your website using S3? (y/N)

Do you want to upload your website using Rackspace Cloud Files? (y/N)

Do you want to upload your website using GitHub Pages? (y/N)
  • Upon completing these questions, some new files and sub folders will be automatically created in your folder, such as content folder, output folder, and fabfile.py, pelicanconf.py, and publishconf.py.

To view the content of your folder, run the following command:

(pelicantest) D:\TDSblog>dir

To view the structure of your folder, run the following command:

(pelicantest) D:\TDSblog>tree

Apply Bootstrap3 Theme and Install Plugins¶

To get the look and feel of the default theme of Pelican-based website, check out this link. If you're interested in other Pelican themes, check out this link.

In this guide, we'll use Bootstrap3 as the website theme.

  • Execute the following command to set up your folder as a git repository:
(pelicantest) D:\TDSblog>git init
  • Execute the following command to clone the Pelican themes repository to your local computer:
(pelicantest) D:\TDSblog>git clone https://github.com/getpelican/pelican-themes/ themes
  • Execute the following command to clone the Pelican plugins repository to your local computer: 
(pelicantest) D:\TDSblog>git clone https://github.com/getpelican/pelican-plugins plugins
  • Add the following information to the bottom of pelicanconf.py in your folder and save. 
MARKUP = ('md', 'ipynb')

PLUGIN_PATHS = ['plugins']
PLUGINS = ['ipynb.markup','i18n_subsites']

THEME = 'themes/pelican-bootstrap3'
JINJA_ENVIRONMENT = {'extensions': ['jinja2.ext.i18n']}
  • Execute the following command to add the Pelican plugins for Jupyter/IPython Notebooks
(pelicantest) D:\TDSblog>git submodule add git://github.com/danielfrg/pelican-ipynb.git plugins/ipynb
  • Upon completing the steps above, you should have additional files and sub folders created in your folder, such as themes folder, plugins folder, and .gitmodules file.

To view the content of your folder, run the following command:

(pelicantest) D:\TDSblog>dir

To view the structure of your folder, run the following command:

(pelicantest) D:\TDSblog>tree

Write the First Blog Post¶

Once you complete installing the Pelican themes and plugins, you can start writing your first blog post using the Jupyter Notebook interface.

  • Create a Jupyter Notebook (a .ipynb file) with any content you'd like and save it to the content folder.

If you're not familiar with Jupyter Notebook, you can download a quick example of .ipynb from here and save it to your content folder.

  • Check out this link for more information about what Jupyter Notebook is about
  • Check out this link for a gallary of interesting Jupyter Notebooks

  • Check out this link for the Markdown Syntax

  • Create a new file with the same name as .ipynb file but with extension .ipynb-meta.

  • Enter the following content into the .ipynb-meta file and save it to your content folder.

Title:

Slug:

Date:

Category:

Tags:

Author:

Summary:

For example, the Jupyter Notebook I created is first-post.ipynb and the metadata file I created is first-post.ipynb-meta with the following content:

Title: First Post

Slug: first-post

Date: 2017-07-09 15:00

Category: posts

Tags: Pelican Python GitHub Jupyter

Author: Superduper Writer

Summary: First blog post on Superduper Blog

.ipynb and .ipynb-meta should be in pairs in the content folder when adding a new post. .ipynb provides the content of your post, such as the charts, words, code etc. and .ipynb-meta provides the metadata of your post.

Generate HTML Files¶

Now you've completed writing the first post and its metadata, we'll generate HTML files and preview the blog before posting it to GitHub. Before starting the following steps, make sure you're still under the main folder, not under content. If not, run pushd yourfolder in the Windows command shell. In my case, I run pushd D:\TDSblog instead.

  • Run the following command to generate HTML files
(pelicantest) D:\TDSblog>pelican content
  • Go to the output folder by executing the following command
(pelicantest) D:\TDSblog>cd output
  • Run the following command to post your blog to local server
(pelicantest) D:\TDSblog>python -m pelican.server
  • Preview the blog by visiting localhost:8000 using your internet browser.

Set Up GitHub Pages¶

Signing up a GitHub account is one of the prerequisites. Once you have a GitHub account, you can follow steps below to set up your website hosted by GitHub on the domain of GitHub.io.

  • Create a repository named as username.github.io where username is the name of your GitHub account. This link provides more details about how to set up a repository on GitHub. Follow steps under the Create A Repo section.

You'll have two options:

  1. A free account with your files visible to any GitHub users
  2. A paid private account

For now, you can start with the free one to set up the bone of your blog. You can upgrade it to paid later on if you'd like.

  • Run the following command to add the new remote repository you just created for your local git repository. Replace username with the name of your GitHub account. Note that you need to run the command under the main folder where your blog files sit.
(pelicantest) D:\TDSblog>git remote add origin git@github.com:username/username.github.io.git

  • Update SITEURL in publishconf.py in your local folder, by setting SITEURL to https://username.github.io. Replace username with the name of your GitHub account.

  • Run the following command to configure Pelican settings in publishconf.py

(pelicantest) D:\TDSblog>pelican content -s publishconf.py
  • Run the following command from your main folder in order to import the content of output folder to the master branch
(pelicantest) D:\TDSblog>ghp-import output -b master
  • Run the following command from your main folder in order to push the content to GitHub
(pelicantest) D:\TDSblog>git push origin master
  • Visit https://username.github.io with username replaced by the name of your GitHub account. You should see your blog running on the GitHub.io domain.

If you make any additional edits to the first post of your blog, follow the steps as below and your GitHub Pages will be updated accordingly:

  1. Save the updated .ipynb and .ipynb-meta file in your content folder and keep both file names the same, except for extensions
  2. Run pelican content -s publishconf.py in the root of the local repository
  3. Run ghp-import output -b master in the root of the local repository
  4. Run git push origin master in the root of the local repository

If you'd like to use a custom domain, i.e. not the GitHub.io domain, check out this link for additional information.
